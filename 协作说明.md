# 协作说明（必读）

## 1. 文件分类与存放规范
请务必将各类文件存放在其对应的专属文件夹中。
根据 `.cursorrules` 全局规则约束：**所有由 AI 生成或人工编写的演示（Demo）文件、示例代码等，必须统一保存至 `Demos/` 文件夹下**，保持根目录整洁。

## 2. 内置技能使用指南

### 2.1 公司规范高保真 Demo 生成
项目中已内置符合公司规范的高保真设计技能文件：`skill_seerdesign_demo_v2.0`。
- **使用方法**：在对话框中输入 `@skill_seerdesign_demo_v2.0`（或选中该文件）后，附上本次具体的需求内容即可。若在需求中说明**“使用 aTrust 导航”**，系统将默认生成带有该导航的 HTML 格式 Demo。
- **使用示例**：
  > `@skill_seerdesign_demo_v2.0` 请使用 aTrust 导航帮我制作一份 Demo。具体需求如下：
  > 
  > 1. **菜单层级**：增加一个一级菜单“GENAI 保护”（位置在“零信任网络访问”下方），该菜单下包含二级菜单“Agent 风险识别”。
  > 2. **页面框架**：选中“Agent 风险识别”后，页头标题显示为“Agent 风险识别”。标题右侧提供两个 Tab：“Agent 风险可视”和“AI 应用分析”，默认选中“Agent 风险可视”。页面整体采用“顶部概览 + 底部基础表格”的布局结构。
  > 3. **内容业务逻辑（Agent 风险可视）**：此页面用于识别企业员工安装的 Agent（如 Open claw、Nanobot 等）及其风险情况。需识别的风险包含：**配置风险**（如公网暴露、未启用认证、未运行在沙箱空间、群聊回复无需@提及等）和**工具风险**（如恶意 Skills、恶意 MCP 等）。
  > 4. **页面展示细节**：
  >    - **顶部数据概览**：统计全企业安装的 Agent 总数、本地桌面运行的数量、存在风险的 Agent 数量（按类型区分）以及全网工具风险的大致数量。
  >    - **底部详细信息列表**：以终端为维度展示详细列表（一行一个）。字段应包括：Agent 类型、安装终端名称、终端归属人、终端 IP、当前 Agent 运行环境（本地桌面 / Agent Space）、存在的配置风险、存在的工具风险，并支持点击操作查看“详情”。

## 3. 项目协作与版本控制

- **修改与备份**：在更改已有的 HTML 文件之前，**请务必先复制一份作为备份**（或直接基于副本进行编辑），避免修改过程出现不可逆的错误导致无法还原原页面。
- **提交规范**：每次修改完毕并确认无误后，必须使用 Git 命令将代码更新并提交到仓库（`git add .` -> `git commit -m "..."` -> `git push`），**提交信息（Commit Message）必须清晰备注本次的具体改动点**。
